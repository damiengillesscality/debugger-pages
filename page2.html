Step 2: Explore the Call Stack

Perfect! You've hit the breakpoint. Set the break point again and let's understand what you're seeing:

üìö Understanding the Call Stack

Look at the Call Stack panel (right side of Sources tab). It shows:

  • strangeFunction - The current function where execution paused
  • setInterval - The timer that called our function
  • init - This exercise inner init function
  • Exercise2Controller - The constructor that track your progress in this exercise, you can easily retro engineer this exercise !

üí° Key Point: The call stack shows how you got to the current point of execution!

üîç Examining Variables

In the Scope panel, you can see:

  • Script variables - Variables in the current script page
  • Global variables - Variables accessible throughout the whole site scripts
  • Hover over variables in the code to see their values
